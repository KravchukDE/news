# Техническое задание: Исследование сервисов генерации изображений

## Контекст

Мы строим автоматический пайплайн публикации новостных дайджестов в Instagram. Нужен сервис для генерации уникальных изображений (1080x1350, формат 4:5) для каждого поста.

## Метод генерации

**Image-to-image (img2img) на базе шаблонов-референсов.**

У нас есть 2-10 заранее созданных шаблонов — они задают стиль, цветовую палитру, композицию. При каждой публикации:

1. Берётся рандомный шаблон
2. Генерируется короткий промпт на основе содержания дайджеста (1 предложение, на английском)
3. Сервис получает шаблон + промпт → возвращает уникальное изображение в том же стиле
4. Текст заголовка накладывается **отдельно, программно** (не генератором!)

Таким образом, от сервиса требуется **только** генерация фонового изображения в стиле референса. Текст на изображении мы НЕ генерируем через AI — он накладывается отдельно.

## Требования к сервису

### Обязательные

1. **API доступ** — программный вызов, без ручного вмешательства
2. **Image-to-image** — приём референсного изображения + текстового промпта
3. **Контроль стиля** — результат должен быть визуально близок к референсу (палитра, настроение, стиль)
4. **Разрешение** — минимум 1080x1350 пикселей (или возможность задать aspect ratio 4:5)
5. **Скорость** — генерация за < 30 секунд
6. **Качество** — изображение должно выглядеть профессионально (не «AI-мусор» с артефактами)
7. **Стоимость** — разумная для 1-3 изображений в день (~30-90 в месяц)

### Желательные

8. **Style reference / style transfer** — возможность задать «стиль как на этом изображении»
9. **Negative prompts** — возможность исключить нежелательные элементы
10. **Seed / детерминизм** — воспроизводимость результатов
11. **Inpainting** — возможность оставить часть изображения (например, фрейм) и генерировать только фон
12. **Batch API** — генерация нескольких вариантов за один вызов

## Сервисы для исследования

### Приоритетные (2025-2026, актуальные)

- **Google Imagen 3 / Gemini** — последняя модель Google, API через Vertex AI
- **Flux (Black Forest Labs)** — Flux Pro / Flux Dev, API через Replicate или BFL напрямую
- **Ideogram v3** — сильный в тексте на изображениях, но нам текст не нужен от него
- **Midjourney API** — если появился публичный API (проверить!)
- **Recraft v3** — сильный в дизайне, поддержка стилей
- **Leonardo.ai** — API, img2img, style reference
- **Stability AI (SDXL, SD3, Stable Cascade)** — открытые модели, Replicate/самохост

### Хостинг-платформы для моделей

- **Replicate** — хостит Flux, SDXL, SD3 и другие, простой API, pay-per-use
- **Together.ai** — быстрый inference, поддерживает Flux
- **fal.ai** — быстрый API для генеративных моделей
- **RunPod** — GPU serverless для самохоста

### Проверить актуальность

- **DALL-E** — какая сейчас актуальная версия? gpt-image-1?
- **Adobe Firefly API** — есть ли публичный API?
- **Canva API** — есть ли img2img?

## Критерии сравнения

Для каждого сервиса собрать:

| Параметр | Что узнать |
|----------|-----------|
| Модель | Название и версия |
| API | URL документации, формат запроса |
| img2img | Поддерживает ли? Какие параметры? |
| Style reference | Можно ли передать изображение как стилевой референс? |
| Разрешение | Максимальное, поддержка aspect ratio |
| Скорость | Среднее время генерации |
| Цена | За 1 изображение, месячный план если есть |
| Качество | Субъективная оценка (примеры если возможно) |
| Ограничения | Content policy, rate limits, региональные ограничения |
| SDK | Node.js / Python, примеры кода |

## Формат результата

Результат исследования должен содержать:

1. **Таблицу сравнения** всех проверенных сервисов
2. **Рекомендацию** — какой сервис использовать и почему
3. **Пример кода** — минимальный рабочий вызов API для рекомендуемого сервиса
4. **Оценку стоимости** — сколько будет стоить 30-90 генераций в месяц
5. **Риски и ограничения** — что может пойти не так

## Ограничения исследования

- Фокус на сервисах с **готовым API** (не на моделях, требующих самохостинга)
- Приоритет на **актуальные модели 2025-2026** (не устаревшие)
- Не исследовать сервисы без img2img / style reference
- Не исследовать сервисы только с Web UI (без API)
